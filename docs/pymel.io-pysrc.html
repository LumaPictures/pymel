  2  """ 
  3  The file command is the most important of the commands that should not be imported into the default namespace because 
  4  it conflicts with python's builtin file class.  Since the file command has so many flags, I decided to kill two birds with 
  5  one stone: by breaking the file command down into multiple functions -- one for each primary flag -- the command becomes 
  6  more readable and also allows its functionality to be found directly within the pymel namespace.    
  7   
  8  for example, instead of this: 
  9           
 10          >>> expFile = cmds.file( exportAll=1, preserveReferences=1 ) 
 11           
 12  you can do this: 
 13   
 14          >>> expFile = exportAll( preserveReferences=1) 
 15           
 16  some of the new commands were changed slightly from their flag name to avoid name clashes and to add to readability: 
 17   
 18          >>> importFile( expFile ) 
 19          >>> createReference( expFile ) 
 20   
 21  also note that the 'type' flag is set automatically for you when your path includes a '.mb' or '.ma' extension.
